# Poll CRUD Operations

This document explains the poll management endpoints, access control rules, and data model.

---

## Overview

Polls follow a lifecycle with three statuses:

```
  ┌───────┐    publish    ┌──────┐    close/expire    ┌────────┐
  │ DRAFT │ ────────────► │ OPEN │ ──────────────────► │ CLOSED │
  └───────┘               └──────┘                     └────────┘
      │                                                     
      │ delete                                              
      ▼                                                     
  (removed)                                                 
```

- **Draft** — editable, deletable, not visible to other users.
- **Open** — accepting votes, visible based on `visibility` setting. *(Publish in Phase 2)*
- **Closed** — voting ended, results available. *(Close in Phase 2)*

## Poll Document Schema

```json
{
  "_id": ObjectId("..."),
  "creator_id": ObjectId("..."),
  "title": "What's your favourite language?",
  "description": "Pick one!",
  "poll_type": "single_choice",
  "status": "draft",
  "visibility": "public",
  "options": [
    { "id": "opt_1", "label": "Python" },
    { "id": "opt_2", "label": "TypeScript" },
    { "id": "opt_3", "label": "Go" }
  ],
  "results_visibility": "always",
  "expires_at": null,
  "created_at": ISODate("2024-06-04T12:00:00Z"),
  "updated_at": ISODate("2024-06-04T12:00:00Z"),
  "published_at": null,
  "closed_at": null
}
```

### Field Reference

| Field                | Type           | Description                              |
| -------------------- | -------------- | ---------------------------------------- |
| `_id`                | ObjectId       | Auto-generated by MongoDB                |
| `creator_id`         | ObjectId       | Reference to the user who created it     |
| `title`              | string         | Poll title (1-300 chars)                 |
| `description`        | string \| null | Optional description (max 2000 chars)    |
| `poll_type`          | enum           | `single_choice` or `multi_choice`        |
| `status`             | enum           | `draft`, `open`, or `closed`             |
| `visibility`         | enum           | `public` or `private`                    |
| `options`            | array          | 2-10 options with `id` and `label`       |
| `results_visibility` | enum           | `always`, `after_voting`, `creator_only` |
| `expires_at`         | datetime\|null | Optional auto-close timestamp            |
| `created_at`         | datetime       | Creation timestamp                       |
| `updated_at`         | datetime       | Last modification timestamp              |
| `published_at`       | datetime\|null | Set when status changes to `open`        |
| `closed_at`          | datetime\|null | Set when status changes to `closed`      |

---

## Option ID Generation

Option IDs are auto-generated sequentially when a poll is created or updated:

```
Input:  [{"label": "Python"}, {"label": "Go"}, {"label": "Rust"}]
Output: [{"id": "opt_1", "label": "Python"}, {"id": "opt_2", "label": "Go"}, {"id": "opt_3", "label": "Rust"}]
```

This ensures consistent, predictable IDs for vote tracking.

## Endpoints

### 1. Create Poll (`POST /api/v1/polls`)

**Auth:** Required (Bearer token)

#### Request Body

```json
{
  "title": "Favourite Language?",
  "description": "Choose your favourite programming language",
  "poll_type": "single_choice",
  "visibility": "public",
  "options": [
    { "label": "Python" },
    { "label": "TypeScript" },
    { "label": "Go" }
  ],
  "results_visibility": "always",
  "expires_at": "2024-12-31T23:59:59Z"
}
```

#### Validation Rules

| Field                | Rule                                    |
| -------------------- | --------------------------------------- |
| `title`              | Required, 1-300 chars                   |
| `description`        | Optional, max 2000 chars                |
| `poll_type`          | `single_choice` or `multi_choice`       |
| `visibility`         | `public` or `private`                   |
| `options`            | Required, 2-10 items, each 1-200 chars  |
| `results_visibility` | `always`, `after_voting`, `creator_only` |
| `expires_at`         | Optional ISO 8601 datetime              |

#### Behaviour

- Poll is always created with `status = "draft"`.
- `creator_id` is set from the authenticated user's ID.
- `created_at` and `updated_at` are set to current UTC time.
- `published_at` and `closed_at` are set to `null`.
- Option IDs are auto-generated (`opt_1`, `opt_2`, …).

#### Response (`201 Created`)

Full `PollResponse` object.

---

### 2. List My Polls (`GET /api/v1/polls/my`)

**Auth:** Required

#### Query Parameters

| Param    | Type   | Default | Description                      |
| -------- | ------ | ------- | -------------------------------- |
| `status` | string | —       | Filter: `draft`, `open`, `closed` |
| `page`   | int    | 1       | Page number (≥ 1)                |
| `limit`  | int    | 20      | Items per page (1-100)           |

#### Behaviour

- Returns only polls where `creator_id` matches the current user.
- Sorted by `created_at` descending (newest first).
- Paginated response with metadata.

#### Response (`200 OK`)

```json
{
  "polls": [ /* PollResponse objects */ ],
  "total": 42,
  "page": 1,
  "limit": 20,
  "pages": 3
}
```

---

### 3. Get Poll (`GET /api/v1/polls/{poll_id}`)

**Auth:** Required

#### Access Control Rules

A user can view a poll if **any** of these conditions are true:

1. **Creator** — the user created the poll.
2. **Public + Open** — the poll's visibility is `public` AND status is `open`.
3. **Active Invitation** — the user has a pending invitation to the poll. *(Full invitation system in Phase 2)*

#### Error Responses

| Status | Condition                    | Detail                               |
| ------ | ---------------------------- | ------------------------------------ |
| 404    | Poll ID does not exist       | "Poll not found"                     |
| 404    | Invalid ObjectId format      | "Poll not found"                     |
| 403    | No access (private/draft)    | "You do not have access to this poll" |

---

### 4. Update Poll (`PATCH /api/v1/polls/{poll_id}`)

**Auth:** Required (creator only)

#### Restrictions

- Only polls in **draft** status can be updated.
- Only the **creator** can update a poll.

#### Updatable Fields

All fields are optional — only provided fields are updated:

| Field                | Behaviour                            |
| -------------------- | ------------------------------------ |
| `title`              | Replace title                        |
| `description`        | Replace description                  |
| `poll_type`          | Change to single/multi choice        |
| `visibility`         | Change public/private                |
| `options`            | Replace all options (re-generates IDs) |
| `results_visibility` | Change results visibility            |
| `expires_at`         | Set or change expiration             |

`updated_at` is automatically set to current UTC time on every update.

#### Error Responses

| Status | Condition           | Detail                                       |
| ------ | ------------------- | -------------------------------------------- |
| 404    | Poll not found      | "Poll not found"                             |
| 403    | Not the creator     | "Only the poll creator can update this poll"  |
| 409    | Not in draft status | "Only polls in draft status can be updated"   |

---

### 5. Delete Poll (`DELETE /api/v1/polls/{poll_id}`)

**Auth:** Required (creator only)

#### Restrictions

- Only polls in **draft** status can be deleted.
- Only the **creator** can delete a poll.
- This is a **hard delete** — the document is permanently removed.

#### Response

`204 No Content` (empty body)

#### Error Responses

| Status | Condition           | Detail                                       |
| ------ | ------------------- | -------------------------------------------- |
| 404    | Poll not found      | "Poll not found"                             |
| 403    | Not the creator     | "Only the poll creator can delete this poll"  |
| 409    | Not in draft status | "Only polls in draft status can be deleted"   |

---

### 6. Publish Poll (`POST /api/v1/polls/{poll_id}/publish`)

**Auth:** Required (creator only)

#### Behaviour

- Transitions poll from `draft` → `open`.
- Sets `published_at` to current UTC time.
- Sets `updated_at` to current UTC time.
- After publishing, the following fields become **immutable**:
  - `title`
  - `options`
  - `poll_type`
  - `visibility`

#### Error Responses

| Status | Condition           | Detail                                         |
| ------ | ------------------- | ---------------------------------------------- |
| 404    | Poll not found      | "Poll not found"                               |
| 403    | Not the creator     | "Only the poll creator can publish this poll"   |
| 409    | Not in draft status | "Only polls in draft status can be published"   |

---

### 7. Close Poll (`POST /api/v1/polls/{poll_id}/close`)

**Auth:** Required (creator only)

#### Behaviour

- Transitions poll from `open` → `closed`.
- Sets `closed_at` to current UTC time.
- Sets `updated_at` to current UTC time.
- No more votes can be submitted or withdrawn after closing.

#### Error Responses

| Status | Condition          | Detail                                       |
| ------ | ------------------ | -------------------------------------------- |
| 404    | Poll not found     | "Poll not found"                             |
| 403    | Not the creator    | "Only the poll creator can close this poll"   |
| 409    | Not in open status | "Only polls in open status can be closed"     |

---

### 8. Auto-Close Expired Polls (APScheduler Job)

A background job runs every **60 seconds** and automatically closes polls that have expired.

#### Query Criteria

```javascript
{
  status: "open",
  expires_at: { $lte: now, $ne: null }
}
```

#### Update Applied

```javascript
{
  $set: {
    status: "closed",
    closed_at: now,
    updated_at: now
  }
}
```

- Logs the count of auto-closed polls each run.
- Uses `update_many` for efficient batch processing.

---

### 9. Public Feed (`GET /api/v1/polls/feed`)

**Auth:** Required

Returns a paginated list of public, open polls.

#### Query Parameters

| Param        | Type   | Default      | Description                                    |
| ------------ | ------ | ------------ | ---------------------------------------------- |
| `poll_type`  | string | `all`        | Filter: `single_choice`, `multi_choice`, `all` |
| `sort_by`    | string | `created_at` | Sort field: `created_at` or `expires_at`       |
| `sort_order` | string | `desc`       | Sort direction: `asc` or `desc`                |
| `page`       | int    | 1            | Page number (≥ 1)                              |
| `limit`      | int    | 20           | Items per page (1-50)                          |

#### Response (`200 OK`)

```json
{
  "items": [ /* PollResponse objects */ ],
  "total": 42,
  "page": 1,
  "limit": 20,
  "has_next": true
}
```

## Enumerations Reference

### `poll_type`
| Value           | Description                    |
| --------------- | ------------------------------ |
| `single_choice` | User can select exactly one option |
| `multi_choice`  | User can select multiple options  |

### `status`
| Value    | Description                          |
| -------- | ------------------------------------ |
| `draft`  | Not yet published, editable          |
| `open`   | Accepting votes                      |
| `closed` | Voting ended, results available      |

### `visibility`
| Value     | Description                               |
| --------- | ----------------------------------------- |
| `public`  | Visible to all authenticated users        |
| `private` | Visible only to creator and invited users |

### `results_visibility`
| Value          | Description                                   |
| -------------- | --------------------------------------------- |
| `always`       | Results visible at any time                   |
| `after_voting` | Results visible only after the user has voted |
| `creator_only` | Results visible only to the poll creator      |
